## UC-01 Просматривать объявления

| Поле | Описание |
|------|----------|
| **Вариант использования** | Просматривать объявления |
| **Область действия** | Веб-приложение сервиса объявлений |
| **Уровень** | Цели пользователя |
| **Основное действующее лицо** | Гость или Клиент |
| **Участники** | Отсутствуют |
| **Предусловие** | Пользователь открыл приложение |
| **Гарантии успеха** | Пользователь просмотрел детальную страницу объявления |
| **Триггер** | В любой момент, когда пользователь находится на главной странице |
| **Описание** | 1. Пользователь открывает ленту объявлений. 2. Система отображает список активных объявлений с пагинацией. 3. Пользователь выбирает объявление. 4. Система отображает детальную страницу с описанием, фото и похожими объявлениями. |
| **Расширение** | 2.1. Объявлений нет — система отображает пустое состояние с предложением изменить фильтры. |

## UC-03 Зарегистрироваться

| Поле | Описание |
|------|----------|
| **Вариант использования** | Зарегистрироваться |
| **Область действия** | Веб-приложение сервиса объявлений |
| **Уровень** | Цели пользователя |
| **Основное действующее лицо** | Гость |
| **Участники** | Отсутствуют |
| **Предусловие** | Пользователь не авторизован |
| **Гарантии успеха** | Создан аккаунт, пользователь авторизован |
| **Триггер** | Гость нажимает кнопку «Зарегистрироваться» |
| **Описание** | 1. Гость открывает форму регистрации. 2. Заполняет email, пароль и имя. 3. Система проверяет корректность данных. 4. Система создаёт аккаунт и профиль пользователя. 5. Система отправляет письмо с подтверждением email. 6. Гость переходит по ссылке из письма. 7. Система авторизует пользователя и перенаправляет в личный кабинет. |
| **Расширение** | 3.1. Данные не прошли валидацию — система подсвечивает ошибки, возврат к шагу 2. 4.1. Email уже зарегистрирован — система сообщает об ошибке, возврат к шагу 2. 6.1. Письмо не пришло — пользователь запрашивает повторную отправку, возврат к шагу 5. |
